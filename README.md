# プログラミング練習（初学者向け）

文法を覚えるためだけの教材ではありません。

> **データを持ち、加工し、条件によって処理を変えられるようになる**

これがゴールです。その土台ができたら、Express / React / Next.js へ進めます。

---

## この教材の考え方

- **TypeScript から始めない** … まずは JavaScript として書く。型は「設計」の話なので後回し。
- **フロントから始めない** … React・JSX・npm・Vite を一度に学ぶのは大変。まずは `if` `for` 配列 変数。
- **コンソールアプリ中心** … Node.js で実行し、**VS Code のデバッガで「変数の変化」を目で見る**。
- ファイルは `.ts` だが、最初は型を付けず JavaScript のように書く（型の導入はゴルフゲーム完成後）。

---

## 実行方法

どの課題も `npx tsx` で実行します。

```bash
npx tsx src/01-fizzbuzz.ts
```

`package.json` にショートカットも用意してあります。

```bash
npm run 01   # FizzBuzz
npm run 02   # クイズゲーム
npm run 03   # 価格計算
npm run 04   # 配列（合計・最大値）
npm run 05   # 名前検索
npm run 06   # 最年長者
npm run 07   # ゴルフゲーム（最終課題）
```

## VS Code のデバッガを使う（いちばん大事）

この教材では「コードを読む」よりも **「変数がどう変わるかを見る」** ことを重視します。

1. `src/` の中のファイルを開く（例: `01-fizzbuzz.ts`）
2. 行番号の左をクリックして **ブレークポイント（赤い丸）** を置く
3. **F5** を押す
4. 赤い丸の行で実行が止まり、その時点の変数の中身が見える
5. もう一度 F5（または ▶ボタン）で次へ進む

例えば FizzBuzz の `for` の行で止めると、`i = 1`, `i = 2`, `i = 3` …… と
`i` が変化していく様子を目で追えます。

> 設定は `.vscode/launch.json` に入っています。開いているファイルがそのまま実行されます。

## TypeScript の導入タイミング

**ゴルフゲームが完成してから** で十分です。

最初はこう書きます（型なし。JavaScript と同じ）。

```ts
const clubs = [
  { name: "ドライバー", distance: 250 },
  { name: "アイアン", distance: 120 },
  { name: "パター", distance: 10 },
];
```

慣れてきたら「この形ですよ」と名前（型）を付けます。

```ts
type Club = {
  name: string;
  distance: number;
};

const clubs: Club[] = [ /* ... */ ];
```

型を付けると、`club.nemo` のようなタイプミスを **実行する前に** エディタが教えてくれます。
最初から型を設計しようとしなくて大丈夫。「動くものを作る → 後から型で守る」の順で進めましょう。

`tsconfig.json` は最初ゆるめ（`strict: false`）にしてあります。
型の練習を始めるときに `strict: true` にすると、より細かくチェックしてくれます。
